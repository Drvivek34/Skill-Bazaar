# Contributing to Skill Bazaar

Thanks for helping grow the bazaar! Every skill is welcome as long as it is
**properly attributed** and **correctly placed**.

## Two ways to contribute
1. **Submission form (easiest)** — open a [new submission](https://github.com/Drvivek34/Skill-Bazaar/issues/new/choose) and fill in the form. A maintainer (or an automated agent) will add it.
2. **Pull request** — add the files yourself following the layout below.

## Layout
Each skill lives in its **own sub-folder** inside the most relevant category:

```
<category>/
└── <entry-name-in-kebab-case>/
    ├── README.md     # REQUIRED: what it is, source URL, author, license
    └── ...           # the actual files (a full copy)
```

## Every entry's README.md must include
- **Name** and a one-line description
- **Source**: the original URL
- **Author / credit**
- **License** of the original work
- **Category** it belongs to

## Rules
- Keep entry folders **self-contained**.
- **Attribution is mandatory.** No source = not merged.
- Respect the original license. Don't re-license someone else's work.
- One skill per folder; no dumping zips.
- Use lowercase `kebab-case` for folder names.
- Be respectful — see the [Code of Conduct](CODE_OF_CONDUCT.md).

## Featured
Exceptional, high-quality, or widely-used entries may be mirrored into
`featured/` by maintainers. You can nominate one in your submission.
